Settings

 

 

 

 

Before running a forecast, Users may want to go to the Settings section to set up important parameters.

 

Objective Function:

Precept uses a linear solver to solve the network at every time step. This means some wells may be closed or choked-back to honor process constraints.

The wells that are closed or choked-back depend on the objectives function.

 

For example, on an oil field mostly gas constrained with the objectives to optimize oil production, wells with high GOR will be choked back when gas process constraints are hit.

 

Objective function is a not so easy function to set-up. This is usually done by Cleverfield as part of the initial modeling. Should you wish to change the objective function, we recommand to contact Cleverfield  support@cleverfield.fr

 

 

 

Maintain Voidage

When the checkbox "Play with injection to keep VRR=1 per reervoir" is checked, Precept will bean-back Water Injectors and Gas Injectors with injection potentials in an attempt to maintain a Voidage Replacement Ratio ( VRR =Injection rb / Productiobn rb) of 1 per reservoir. If that option is not activated, injector wells will not be bean-back by Precept (as long as enough fluid is available for injection), which may lead to VRR greater than 1.

 

 

Equal choke opening when constraint is hit

When the checkbox "equal choke opening when constraint is hit" is checked, all wells from the same reservoir are open at the same choke opening (from 0 to 100%). This option needs to be used only on green fields and when all wells from the same reservoir have the same DCA profiles.

 

 

Potentials:

Cut-off can be applied to close wells in future when they become uneconomical.

 

When the option " Apply same potentials to all wells from the same reservoir (explo), all wells from the reservoirs have the same DAC profiles. That means that when users modify one well profile in Decline Curves (DCA) modules, that new profile is automatically copied to other wells from the same reservoir. This option needs to be used for Green Field / Explo only.

 

 

Time step:

By default, Precept adds a time step whenever an event that will impact the forecast occurs. That means a time step will be added when:

- a new well come online,

- a well is converted from producers to injectors, or injectors to producers

- a shutdown starts /ends

- a new constraints is introduced during the forecast.

 

If none of the events above happen, a timestep is created every month or quarter as a minimum (depending on the option chosen in the drop-down menu), unless the input "yearly time step after" is not blank. In that case, a timestep will be created every year as a minimum. Using yearly timestep as opposed to monthly or quarterly timestep improve forecasting speed.

 

At every time step, potentials of wells are calculated from the DCA inputs (see Decline curves: Production and Injection potentials )

 

 

 

Future Losses / Shortfalls

if the "Calculate future losses / Shortfalls" option is activated, Precept will not only calculate future production, but also future shortfalls. Shortfalls are grouped by "Project".

Forecast of Shortfalls aggregated by "Projects" can be visualize in the Results section (see Forecast results: Charts)

 

 

 

Transient production

If that option is activated, transient production/injection will be taken into account, meaning that after a shutdown, a producer will restart at a higher rate, and an injector at a lower rate (due to time needed for bean-up). If that option is not activated, the production before and after the shutdown will be the same. That option requires additional timesteps and therefore takes more time to run. That option should therefore be activated only for short-term forecast. (see Transient)

 

 

 

Voidage

if that option is activated, Precept will calculate production and injection per Patterns at reservoir conditions using the PVT data entered in the PVT section (PVT) and the patterns properties (see Patterns)

 

 

GHG emission:

if that option is activated, Precept will calculate GHG emission, provided that GHG inputs have been entered (GHG emission)

 

 

Economics:

if that option is activated, Precept will calculate basic economics. (see Economical screener)

 

 

What-if scenario:

Users can generate what-if scenario by toggling on/off:

 - projects

 - surface constraints: unconstrained forecast can be generated if that option is off

 - uptime: forecast without uptime can be generated if that option is off.

 

 

Uptime

Uptime can be set at 3 levels.

 - Field level: from the settings input.

 - Manifold level: manifold are nodes directly downstream of a producer, or directly upstream of an injector. Uptime for manifold can be set from the "Constraints changes vs time" interface. ( see Constraints changes vs time )

 - Well level: Uptime for wells can be set from the "Constraints changes vs time" interface. ( see Constraints changes vs time )

 

Therefore:

At every time step, the total uptime for a well is:  Total Uptime for a well  = Field uptime x Manifold Uptime x Well Uptime

 

and:

At every time step: Actual Production = Choke opening x Potential from DCA curves,  with choke opening <= Total Uptime for the well

 

Once Precept solver has solved the network, "choke opening" can be less than "Total uptime" in case a well is process constraints.

Choke opening can be 0 if nodes located downstream are closed.

 

 

Save results option

In order to save time and space on Precept database, the " Save results" option allows to save in the database only selected data.
